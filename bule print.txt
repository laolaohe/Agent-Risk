# AgentRisk 项目蓝图

## An Agent Security Assessment Platform

一句话定义：

> **像 Wiz/Nessus 评估云和主机一样，AgentRisk 用来评估 Agent 的安全风险。**

输入一个 Agent 配置，输出：

* 攻击面（Attack Surface）
* 风险评分（Risk Score）
* 安全建议（Remediation）
* 攻击路径分析（V2+）
* 安全报告

---

# 一、为什么做这个项目

### 行业现状

大家都在研究：

* Prompt Injection
* Agentjacking
* MCP攻击
* Memory Poisoning
* Tool Abuse
* Mythos级AI攻击者

但是企业真正的问题是：

> 我要上线一个Agent，它安全吗？

没人能系统回答。

---

### 定位

不是做一个 Prompt Injection 检测器，也不是做一个 Skill 扫描器。

而是：**Agent Security Assessment Platform**

# 三、核心理念

## Agent = 一个系统

不是"一个 Prompt"，而是：

```
LLM + Prompt + Skill + Tool + MCP + Memory + Workflow + Permission
```

这些共同构成攻击面。

---

# 四、安全模型（项目最核心的创新）

```
Agent
├── LLM
├── System Prompt
├── Skill
├── Tool
├── MCP
├── Memory
├── Workflow
└── Permission
```

每个层都有：Asset / Trust Boundary / Attack Surface / Risk

例如：

**Skill**
- 资产：文件系统、Shell
- 风险：Command Execution、Data Exfiltration

**MCP**
- 资产：Github、Jira、Confluence
- 风险：Supply Chain、Privilege Abuse

**Workflow**
- 资产：Database、Email
- 风险：Sensitive Data Leak

## 输入

手动编写 Skill 文件（Markdown 格式），描述一个 Agent Skill 的能力和权限。也支持 YAML 全量配置。

## V1 检测维度（11 维，三层框架，LLM 内置）

### 第一层：能力测绘
| 维度 | 检测项 |
|------|--------|
| **① 能力枚举** | 列出 Skill 全部能力（shell/http/fs/db/email/browser 等） |
| **② 权限分析** | 每项能力的权限宽度（无限制/受限/最小）和副作用等级（只读/修改/删除/执行） |
| **③ 高危标记** | 单独标记 shell / exec / http / fs_write / db_write / payment / ssh / browser / email |
| **⑨ 外部依赖** | 第三方 API / SaaS / MCP Server / 内部系统 / Docker |

### 第二层：风险识别
| 维度 | 检测项 |
|------|--------|
| **④ 权限组合** | read+http=exfil, shell+fs=RCE, browser+email=phishing, db+email=data_breach |
| **⑤ 用户输入路径** | user input → skill → dangerous operation，是否无过滤无审批 |
| **⑧ 能力一致性** | 描述 vs 权限字段 vs 实际能力（隐藏能力/参数劫持/夸大限制/作用域虚标） |

### 第三层：Skill 特有风险
| 维度 | 检测项 |
|------|--------|
| **🅐 注入指令** | 描述中的指令性语言（供应链注入攻击面） |
| **🅑 审批机制** | 无审批 / 隐式审批 / 明确审批 / 人工审批 |
| **🅒 元数据矛盾** | 权限字段与描述文本互相矛盾 |
| **🅓 凭证硬编码** | API Key / Token / 密码 / 私钥 直接写在 Skill 中 |
| **🅔 作用域穿透** | 路径遍历 / 命令注入 / 通配符滥用 / 环境变量注入 |

## 技术实现

- **分析引擎**：deepseek-chat（兼容 OpenAI SDK，专为 tool use 场景）
- **结构化输出**：Function Calling 强制返回 JSON（含能力清单 + 扩展字段）
- **JSON 容错**：截断修复 + 补全逻辑
- **评分模型**：CVSS 风格三维加权（Exploitability × Impact × Scope）
- **输出**：Rich 彩色终端报告（中文，含能力清单）

## 风险评分引擎（已锁定）

CVSS 风格三维模型：Exploitability × Impact × Scope → Risk Score (0-100)

## 输出

```bash
agentrisk skill scan samples/skill_malicious_shell.md
```

终端 Rich 彩色报告：

```
==== AgentRisk Skill 扫描报告 ====
  文件        samples/skill_malicious_shell.md
  风险评分    #################### 100/100
  风险等级    严重
  整体评语    该 Skill 是一个完全无防护的 Shell 执行后门…

  [1] [严重] 任意 Shell 命令执行（完全 RCE）
      类别: 命令执行
      评分向量: 利用难度=高 / 影响程度=高 / 作用范围=高
      置信度: 100%
      匹配内容: "passes it to /bin/bash -c. No sanitization…"
      >> 攻击链:
         攻击者通过提示注入 → Agent 将命令传递给 /bin/bash -c
         → 攻击者获得宿主系统 RCE → 下载后门、窃取数据、横向移动
      >> 修复建议:
         1. 实施命令白名单  2. 沙箱隔离  3. 强制人工审批

  风险汇总: 严重: 2  高危: 3
============================================================
```
