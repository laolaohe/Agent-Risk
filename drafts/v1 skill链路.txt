  用户敲命令
      │
      │  $ agentrisk skill scan samples/evil.md

      ▼

    │

    │                                                │

第 3 层：LLM Analyzer   analyzers/llm.py
═══════════════════════════════════════════════════════

  analyze_with_llm(skill_text, skill_name)
    │
    ├─ ① 获取 API Key
    │     api_key = DEEPSEEK_API_KEY || OPENAI_API_KEY
    │     未设置 → raise ValueError
    │
    ├─ ② 初始化 DeepSeek 客户端
    │     OpenAI(api_key, base_url="https://api.deepseek.com")
    │
    ├─ ③ 组装 System Prompt（中文 ~500 字）
    │     ┌──────────────────────────────────┐
    │     │ 你是 Agent 安全审计专家           │
    │     │                                  │
    │     │ 5 大检测维度：                   │
    │     │ 1. 命令执行（Shell/bash/exec）   │
    │     │ 2. 文件操作（write/save/delete） │
    │     │ 3. 网络外连（http/curl/fetch）   │
    │     │ 4. 权限控制（scope/approval）    │
    │     │ 5. 注入风险（暗藏指令性语言）     │
    │     │                                  │
    │     │ 评分要求：                       │
    │     │ exploitability × impact × scope  │
    │     │ 三维评分 + severity 判定          │
    │     │                                  │
    │     │ 输出要求：                       │
    │     │ - 全中文 title + description    │
    │     │ - 攻击链逐步还原                │
    │     │ - 可操作的修复建议              │
    │     │ - confidence 置信度 0-1         │
    │     │ - 不误报（区分正常/恶意）        │
    │     └──────────────────────────────────┘
    │
    ├─ ④ 组装 User Prompt
    │     ┌──────────────────────────────────┐
    │     │ ## Skill 名称                    │
    │     │ evil                              │
    │     │                                  │
    │     │ ## Skill 描述全文                │
    │     │ ```                              │
    │     │ (Skill 文件的完整内容)            │
    │     │ ```                              │
    │     │                                  │
    │     │ 请全面审计，调用 report_findings  │
    │     └──────────────────────────────────┘
    │
    ├─ ⑤ 调用 API
    │     client.chat.completions.create(
    │       model="deepseek-v4-pro",
    │       messages=[system, user],
    │       tools=[report_findings],  ← Function Calling 强制 JSON
    │       temperature=0.1           ← 低温度保证一致
    │     )
    │
    │     report_findings 输出结构：
    │     {
    │       "findings": [{
    │         severity, category, title, description,
    │         exploitability, impact, scope,
    │         attack_chain, remediation,
    │         matched_context, confidence
    │       }],
    │       "overall_assessment": "..."
    │     }
    │
    ├─ ⑥ 解析返回
    │     ┌─ 优先：message.tool_calls[0].function.arguments
    │     │  → JSON.parse → {findings, overall_assessment}
    │     │
    │     └─ 兜底：message.content 中提取 JSON 块
    │        → 找 ```json...``` → JSON.parse
    │
    └─ ⑦ 字典 → Finding Pydantic 对象
          for item in raw:
            Finding(
              severity=RiskSeverity(...),
              category=RiskCategory(...),
              rule_id="LLM",   ← 标记为 LLM 发现
              ...
            )

## 数据模型

  Finding                             SkillReport
  ────────                            ────────────
  severity: 严重/高危/中危/低危        skill_file: 文件路径
  category: 8 种风险类别               overall_score: 0-100
  rule_id: "LLM"                      severity: 风险等级
  title: 中文标题                      findings: Finding[]
  description: 中文描述                total_critical / _high / _medium / _low
  exploitability: 高/中/低            overall_assessment: LLM 评语
  impact: 高/中/低
  scope: 高/中/低
  attack_chain: 攻击链
  remediation: 修复建议
  matched_context: 原文可疑片段
  confidence: 0.0~1.0 置信度
