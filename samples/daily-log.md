# Daily Log & Check-in

Auberon 的日常记录系统。统一存储日常活动日志和每日打卡复盘。

## 存储位置

```
~/.openclaw/workspace/life/.daily-logs/
├── 2026-05-17.md
├── 2026-05-18.md
└── ...
```

## 触发方式

### 日常日志

**触发词：** "记录一下"、"今天做了"、"备忘"、"发生了"

**格式：**
```
[HH:MM] 用户说过的原话，一字不改
```

**规则：**
- **记录用户说过的原话，一字不改，不加任何修饰**
- 只加一个时间戳
- 追加到当天 .daily-logs/YYYY-MM-DD.md

### 每日打卡

**触发词：** "打卡"、"复盘"、"checkin"、"今天回顾"

**固定流程：**

用户说"打卡"后，**直接发送这6个维度让用户回复**（一字不改）：

```
1️⃣ 面试复盘/技术表达库
2️⃣ Web安全/AI4Security
3️⃣ 身体/运动
4️⃣ 项目/工程落地
5️⃣ 行测/考公准备
6️⃣ 岗位追踪/整理

你说就行，说一个我记一个。
```

**操作方式：**
- 用户说出一个维度的内容，立刻记到文件里（说完一个写一个）
- 不用一次性问所有维度，用户说了就记，用户停就停
- 不追问，不催更
- **不能多次追加同一维度的记录** — 每次打卡只在文件里保留一条完整记录，用最新内容替换旧内容

**格式：**
```
## 打卡复盘

- 面试/表达库: xxx
- Web安全/AI4Security: xxx
- 身体: xxx
- 项目: xxx
- 行测/考公: xxx
- 岗位: xxx
```

## 查询

用户问"我昨天做了什么"或"查一下某天的记录"时，从 `.daily-logs/` 读取对应日期文件回复。

## 规则

- 只在用户触发时才记录，**绝不自动写入**
- 不做原话以外的修饰
- 与 memory 系统完全独立
- 不自动清理
